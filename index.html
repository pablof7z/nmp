NMP Pulse — Briefing

2026-07-15 11:30 UTC · state-of-the-project, read top to bottom · ~4 min audio

What NMP is

NMP (Nostr Mobile Primitives) is an embeddable Nostr client engine — a Rust core with Swift and Kotlin SDKs — that owns the machinery apps otherwise rebuild badly: the event store, relay connections, subscription routing, canonical state, signing, and durable publishing. You talk to it with two nouns: a live query (a declarative demand observed through your platform's native reactive primitive) and a write intent (a durable publish obligation tracked through a reattachable receipt). It is a library you call, not a framework you live inside — your app keeps its own architecture, navigation, accounts, and UI.

What it can do today

Reading & state

Declarative live queries with reactive bindings (literal sets, $currentPubkey, derived projections, set algebra). A canonical redb-backed store handles provenance-preserving dedup, replaceable events, NIP-40 expiry (event-driven, no polling), kind:5 deletion with tombstones, and supersession with exact negative-delta invalidation. Snapshots return matching rows plus scoped acquisition evidence — NMP never claims a global synced or complete. EOSE, negentropy, and AUTH are facts about specific planned sources, not proof the network is empty.

Relays & networking

Full connection lifecycle with one finite fan-out ceiling over the whole read plan (local, private, and .onion targets rejected by default). NIP-65 outbox routing is self-bootstrapping: you configure only indexer relays and the engine discovers each author's write/inbox relays via kind:10002. Parse-once typed ingest with bounded parallel signature verification. NIP-11 relay metadata (single-flight, LRU-bounded) and NIP-77 negentropy sync are wired. NIP-42 AUTH participates, but its reserved evidence states are still partial. experimental: AUTH evidence

Signing & identity

Local keys and full NIP-46 remote signing (bunker): independent signer-relay connection, NIP-42 AUTH, request correlation, auth_url/switch_relays, NIP-44 crypto, reconnect across a store close/reopen, and a bounded sign-only path across all four surfaces. Swift projects Primal discovery; Kotlin projects an Android package handoff. gap: Keychain/Keystore providers, NIP-55, per-write identity override, secret zeroization

Publishing

Durable write intents: Accepted is one atomic persistence boundary (frozen body, receipt, and a pending canonical row visible to ordinary queries), then signature promotion, cancellation with compensation, and a persisted outbox that retries with bounded backoff (32 global / 1 per relay, 30s ACK expiry). At-most-once ambiguity is reported as OutcomeUnknown, never a blind resend. Pre-signed events from an external signer publish verbatim.

Protocol modules (opt-in)

NIP-02 following (canonical kind:3, guarded tag-preserving follow/unfollow) on Swift and Kotlin. NIP-29 groups (metadata/membership/moderation; remembered-group browsing via NIP-51 kind:10009). NIP-51 relay-list reading, with guarded favorite-relay editing landing now. An optional content module (plaintext/Markdown, NIP-19, kind:0/NIP-23) plus a SwiftUI component family. not built: NIP-25 reactions, general draft composition, media/Blossom

Storage & diagnostics

Crash-safe redb store with binary canonical rows, secondary + tag + cardinality indexes, and interned relay URLs; an in-memory store for tests; a destructive reset that refuses to delete a live store. Diagnostics are a permanent proof surface: per-relay plan, exact wire filters, connection/AUTH state, EOSE/watermarks, and routing explanation.

Platforms

The Rust core is the truth. The Swift SDK is qualified on the macOS host (XCFramework simulator slices compile; an iOS-Simulator runtime test target is still pending). The Kotlin SDK is a desktop-JVM projection — no Android AAR is qualified yet. iOS/Android device qualification pending

How solid it is

The core is genuinely proven: 100+ Rust test modules plus differential falsifiers against an independent in-memory store, headless outbox/discovery proofs, live-relay tests against purplepag.es and relay.primal.net, and cross-SDK parity tests. CI runs fmt, clippy, workspace tests, Swift and Kotlin package builds, and a trusted-base surface-governance gate that rejects any unaudited change to the public API.

  • An M5 iOS main-thread jank whose root cause is fixed (Swift frame coalescing + Rust indexing + a discovery-churn fix) but not yet re-measured on device with all three live.
  • NIP-29 storage wins are verified as microbenchmarks, not end-to-end device room-open.
  • NIP-42 AUTH evidence states, cross-process store-reset exclusion, and full end-to-end boundedness remain open.

Performance

Designed for bounded memory and streaming, never first-N truncation. Query planning picks one best index and stops at the visible limit: the busiest-room query fell from 5.15 ms to 0.26 ms, a rejected-heavy search from 0.188 ms to 0.005 ms, and a derived-set resolver over a 59,915-row bucket from 3,786 ms to 0.73 ms. The NIP-11 cache has a proven ~67 MiB raw-body ceiling. Native threads are bounded (roughly 7 + 2·relays + 12 tasks), not stream-proportional. The public Rust facade sits at 29,734 lines against its governed 30,000 ceiling — deliberately tight; a proposed history API would need the ceiling raised to 32,000. CI is green.

Where it's heading

Governing the provisional demand/receipt/signer shapes toward a v2 freeze; encoding lifecycle invariants as types instead of conventions (the current sweep); closing platform qualification (an iOS-Simulator test target and an Android AAR); shipping standard secure-storage signer providers; and finishing bounded delivery with explicit shortfall everywhere.
