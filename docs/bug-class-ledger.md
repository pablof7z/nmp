# Bug-class acceptance ledger

This ledger replaces governance-by-lint. A bug class is closed only when a
type/API mechanism makes the bad path unreachable and a falsifier demonstrates
that fact. `TARGET` records an agreed invariant whose mechanism is not built;
`PARTIAL` means current proof covers only part of the promoted contract.

| # | Bug class | Structural exclusion | Current proof status |
|---|---|---|---|
| 1 | **Stale replaceable event retained** | One canonical mutating store door performs exact-id dedup, then replaceable arbitration; no public index setter exists. | **BUILT / M3 verified.** Both stores exercise supersession through the door; retraction of the displaced row is tracked separately. |
| 2 | **Lost or leaked subscription** | Wire demand derives only from refcounted live-query descriptors; there is no public open-REQ verb. NIP-77 acquisition activates a role-separated `REQ {limit:0}` through its exact EOSE before taking the reconciliation snapshot, keeps that live owner open through NEG/backfill, and overlap-closes only after the replacement is proven. | **BUILT / M2 verified; gap-free NIP-77 handoff verified by #563.** Surgical open/close and last-observer withdrawal have headless proofs; boundary, overdelivery, failure, timeout, reconnect, and genuine-relay falsifiers cover the handoff. |
| 3 | **Wrong-relay routing through app relay injection** | Relay selection is compiler output from typed source/lane/context facts. There is no general raw `relays:` argument on observe/publish. | **BUILT for default routing and selector-projected hint/provenance facts (#11); contextual protocol contributions remain TARGET** (§14). |
| 4 | **Uncapped fan-out** | Every plan carries an explicit cap and reports uncovered demand; no relay outside the compiled plan is contacted. | **BUILT for the current engine (#20, #442, #446, #467, #598, #680, #704).** The router applies one deterministic relay ceiling; refused atoms retain `LocalLimit` evidence. Live and charged-retiring relay workers and verifier workers have explicit finite owners bounded by `max_relays`. Access-scoped Public and `Nip42(author)` sessions never share a socket, while reducer-typed read/write admission gives a nonterminal write bounded same-relay time-sharing priority when the physical cap is one (#598): only write ownership may release the same relay's Public worker; protected reads cannot claim that authority; no other relay is evicted; terminal reconciliation restores still-owned Public demand and its exact preamble. A real iOS Simulator falsifier proves route discovery, one publish/ACK, and read restoration at `maxRelays = 1`. **Observation delivery no longer owns an OS thread (#680):** row, window, diagnostics, follow, receipt, and follow-action streams are waker-driven async pull handles over engine-owned bounded mailboxes, so NMP OS-thread count is independent of the number of live observations (a 1,000-handle scaling falsifier proves 0 thread growth) and there is no app-visible native-task capacity — `max_native_tasks`/`maxNativeTasks`/`ExecutorSaturated` are removed from every SDK. Snapshot mailboxes conflate exactly; FIFO fact streams retain a fixed 32-item prefix and surface typed lag, with durable receipts replayed through finite pages using identity-stable continuation state bounded by relay fan-out and consumer close/drop detaching its exact live sink. This bounds live delivery, not total persisted attempt retention, which remains tracked under #46. **Internal capacity refusal is also gone (#704):** one engine-owned async runtime hosts ready-or-pending adapter work; waker-driven completion doors park no worker thread, and no operation can be refused because an internal scheduler is occupied. Physical resources remain bounded by private backpressure: NIP-11 admits at most 8 distinct HTTP/DNS/body flights and coalesces same-relay callers onto one shared completion, while each NIP-46 session uses finite request/control/event queues plus independent cancellation/shutdown control. NIP-46 sessions share async execution and retain only their bounded per-session transport workers, which preserve each session's distinct NIP-42 transport identity; foreign completions whose contract permits blocking run on fresh per-operation threads rather than an admitted internal pool. `docs/design/durable-write-signing-and-retry.md`, `docs/design/async-observation-handles.md`, and `docs/design/internal-executor-elimination.md` record the current architecture; `native-task-executor.md` is retained as the superseded record. |

| 8 | **Assuming NIP-77 support** | Negentropy requires a capability token minted only after behavioral probe; NIP-11 advertisement is tracked separately and can never mint the token. Unprobed relays use REQ. | **BUILT / M3 verified; NIP-11 evidence split verified by #185.** |
| 9 | **Accepted/enqueued treated as converged** | Durable `Accepted` is constructible only after atomic persistence of intent, receipt, frozen body, and canonical pending row. Live fact delivery is a fixed-capacity FIFO with typed lag, finite-page durable replay uses an identity-stable per-lane continuation so mutation between pages neither skips nor duplicates evidence, and consumer close/drop deterministically removes its identity-bound live sink even if the receipt never emits again. `OutcomeUnknown` projects truthfully across FFI/Swift/Kotlin, while native stable-id reattachment remains. |
| 10 | **Signer drift after acceptance** | Publish defaults to the signer for `$currentPubkey`, permits an explicit identity override, then freezes the selected expected author at `Accepted`. | **PARTIAL.** Default-author freezing, exact-pubkey restart resume, and a real delayed NIP-46 reattachment are built; explicit per-write override and standard platform vault providers remain. |
| 11 | **App owning interest expansion** | `Derived`/`SetOp` resolve inside the engine; helpers return the same closed printable graph; no expanded-set callback exists. | **BUILT for the raw grammar.** Module-provided reusable fragments remain TARGET. |
| 12 | **Presentation or plaintext policy in core** | Engine/module values contain protocol-semantic raw data only. Formatting remains app/UI-owned; decrypted results, when supported, remain raw protocol values. | **PARTIAL.** Unencrypted raw-token boundary exists; decrypt return path and protocol modules are not complete. |
| 13 | **Late-arriving event skipped by presentation pagination** | Windowing is a policy on the one read noun: `observe(query, window)` with an engine-owned expandable window maintains the complete bounded canonical partition ordered by the exact exclusive `(created_at DESC, event_id ASC)` cursor. Growth is the declarative, monotonic `request_rows(at_least:)`; the host never constructs, holds, or replays a protocol cursor or continuation token. Acquisition cursors remain separate engine state. | **BUILT across Rust, UniFFI, Swift, and Kotlin (#474, resurfaced per #485).** Same-timestamp rows advance with exact exclusive ordering across multiple pages; newer/late arrivals, retractions, replacement, expiry, compensation, and provenance growth rebalance one atomic row set. Delivery mode derives from boundedness (unbounded ⇒ exact rebased deltas whose intermediate reducer emits may conflate, windowed ⇒ conflated authoritative snapshots), `AtBound` is a mechanical frame fact rather than an error, store failure preserves the last rows, and cancellation withdraws the session exactly once. | 25 | **Engine configuration defaults drift across Rust and native SDK entry points** | One declarative include projects every app-facing engine default into the transport/runtime constants, direct-Rust `Default`, and UniFFI's literal-bearing record/default. Generated Swift/Kotlin records are zero-argument constructible, and both ergonomic wrappers read omitted numeric values from that generated boundary. The `nmp-ffi` build script rejects any reintroduced numeric owner in those projections before compilation continues. | **BUILT by #728/#736.** Rust proves `EngineConfig::default()` and `NmpEngineConfig::default()` field parity; Swift and Kotlin compare their complete ergonomic defaults to a generated `NmpEngineConfig()` and separately preserve explicit-zero semantics. Every status must remain honest. A design document, issue, or passing adjacent
test is not proof. Promoting `TARGET`/`PARTIAL` to `BUILT` requires a falsifier
against the supported public facade and, where applicable, restart and platform
projection tests.

PR review applies this same rule prospectively, before a new bug class ever
needs a row here: see `docs/design/architecture-review-gates.md` for the six
review gates (Noun, Reachability, Bool-Lifecycle, Destructive-API, Cross-SDK
Parity, Falsifier-honesty) this ledger's rule expands into at review time.
