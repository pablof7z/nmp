# Bug-class acceptance ledger

This ledger replaces governance-by-lint. A bug class is closed only when a
type/API mechanism makes the bad path unreachable and a falsifier demonstrates
that fact. `TARGET` records an agreed invariant whose mechanism is not built;
`PARTIAL` means current proof covers only part of the promoted contract.

| # | Bug class | Structural exclusion | Current proof status |
|---|---|---|---|
| 1 | **Stale replaceable event retained** | One canonical mutating store door performs exact-id dedup, then replaceable arbitration; no public index setter exists. | **BUILT / M3 verified.** Both stores exercise supersession through the door; retraction of the displaced row is tracked separately. |
| 2 | **Lost or leaked subscription** | Wire demand derives only from refcounted live-query descriptors; there is no public open-REQ verb. | **BUILT / M2 verified.** Surgical open/close and last-observer withdrawal have headless proofs. |
| 3 | **Wrong-relay routing through app relay injection** | Relay selection is compiler output from typed source/lane/context facts. There is no general raw `relays:` argument on observe/publish. | **BUILT for default routing and selector-projected hint/provenance facts (#11); contextual protocol contributions remain TARGET** (§14). |
| 4 | **Uncapped fan-out** | Every plan carries an explicit cap and reports uncovered demand; no relay outside the compiled plan is contacted. | **BUILT for the current engine (#20, #442, #446, #467).** The router applies one deterministic relay ceiling; refused atoms retain `LocalLimit` evidence. Live and charged-retiring relay workers, verifier workers, native observer/action tasks, signer waiters/mappers, NIP-11 flights/waiters, and NIP-46 forwarders all have explicit finite owners. `max_native_tasks` is a zero-queue admission bound: native bridges start before stream/write ownership transfers, saturation is typed separately from OS spawn refusal, and accepted signer obligations remain retryable. NIP-11 additionally shares one immutable document payload across each cache/flight/waiter fan-out, so its 64-waiter bound multiplies pointers rather than raw bodies. Relay and native-task slots release only after owned reapers join the OS threads. Exact event-driven census, callback-shutdown, forgotten-reservation, injected-refusal, Rust/FFI, and Swift/Kotlin lifecycle falsifiers cover the boundary; `docs/design/native-task-executor.md` records the honest thread formula and NIP-46 qualification. |
| 5 | **Dedup or provenance loss** | Duplicate id insertion merges provenance before other processing; event id/signature bytes remain canonical. | **BUILT / M3 verified** for both store backends and live duplicate delivery. |
| 6 | **Private-event republish** | Private/narrow route values have no widen operation; unresolved private routing fails closed. | **BUILT for current narrow route types; future protocol modules must preserve it.** |
| 7 | **Cache evidence presented as global truth** | Query snapshots expose rows plus per-current-plan source facts and explicit shortfalls, computed over every root and interior atom. No public query state expresses a global completeness verdict. | **BUILT for scoped evidence and the #12 interior-atom proof; PARTIAL for broader #49.** Rust/FFI/Swift/Kotlin carry `AcquisitionEvidence`, while full `SourceAuthority + AccessContext` descriptor identity and context-isolated evidence/coalescing remain TARGET. |
| 8 | **Assuming NIP-77 support** | Negentropy requires a capability token minted only after behavioral probe; NIP-11 advertisement is tracked separately and can never mint the token. Unprobed relays use REQ. | **BUILT / M3 verified; NIP-11 evidence split verified by #185.** |
| 9 | **Accepted/enqueued treated as converged** | Durable `Accepted` is constructible only after atomic persistence of intent, receipt, frozen body, and canonical pending row. | 10 | **Signer drift after acceptance** | Publish defaults to the signer for `$currentPubkey`, permits an explicit identity override, then freezes the selected expected author at `Accepted`. | **PARTIAL.** Default-author freezing, exact-pubkey restart resume, and a real delayed NIP-46 reattachment are built; explicit per-write override and standard platform vault providers remain. |
| 11 | **App owning interest expansion** | `Derived`/`SetOp` resolve inside the engine; helpers return the same closed printable graph; no expanded-set callback exists. | **BUILT for the raw grammar.** Module-provided reusable fragments remain TARGET. |
| 12 | **Presentation or plaintext policy in core** | Engine/module values contain protocol-semantic raw data only. Formatting remains app/UI-owned; decrypted results, when supported, remain raw protocol values. | **PARTIAL.** Unencrypted raw-token boundary exists; decrypt return path and protocol modules are not complete. |
| 13 | **Late-arriving event skipped by presentation pagination** | One engine-owned history session maintains the complete bounded canonical partition; its opaque exclusive continuation is bound to the engine epoch, session, full demand descriptor, generation, and exact `(created_at, event_id)` boundary. Acquisition cursors remain separate engine state and cannot be constructed or advanced by the host. | **BUILT across Rust, UniFFI, Swift, and Kotlin (#474).** Same-timestamp rows advance with exact exclusive ordering across multiple pages; newer/late arrivals, retractions, replacement, expiry, compensation, and provenance growth rebalance one atomic row set. Wrong-version/engine/session/descriptor/generation continuations fail by typed axis, store failure preserves the last rows, cancellation withdraws the session, restart rejects the process-local continuation, and store/runtime/native-wrapper falsifiers prove bounded scans, state, and latest-wins delivery. Durable cursor resume and a global end verdict are deliberately not claimed. |
| 14 | **Schema ownership confused with contextual publication authority** | A module claims only exact NIP-defined schemas. Contextual operations contribute closed typed tags/routes without claiming a foreign draft's kind. | **TARGET.** Existing ownership design incorrectly gates every route contribution on kind ownership. |
| 15 | **Pending write bypasses normal query semantics** | `Pending(intentId) | Signed(signature)` is state on the canonical store row; acceptance, promotion, cancellation, replacement, delete, and expiry all use the one store mutation path. | **BUILT.** Pending visibility and restart/cancellation use the ordinary store/resolver graph; no optimistic overlay exists. |
| 16 | **Durable obligation lost or duplicated by retry layers** | Outbox is the sole persisted `(intent, relay)` attempt owner; transport owns sockets only; signer adapter owns one RPC; one deadline scheduler owns time. | **BUILT.** Versioned exact-byte ordinal/outcome ownership, stable eligibility ordering, deterministic backoff, global/per-relay caps, deadline consumption, offline/AUTH suspension, and Durable/AtMostOnce restart behavior are falsified. Rust/FFI/Swift/Kotlin receipts project waiting, eligibility, ambiguous handoff, and persisted written-handoff truth without collapsing queue acceptance into `Sent`. |
| 17 | **Silent truncation under limits** | Every graph/wire/relay/result limit either preserves exact semantics, rejects explicitly, or emits shortfall evidence. First-N masquerading as complete is unrepresentable. | **PARTIAL overall; relay fan-out is BUILT (#20) and projected-id wire chunking is BUILT (#11).** Whole-demand relay refusals retain exact contextual `LocalLimit` evidence; projected singleton ids widen-only pack into filters capped at 256 and overflow into additional exact chunks. Other graph, result, and presentation limits still need the same end-to-end proof. |
| 18 | **Demand/evidence conflated across source or access context** | Descriptor identity is `selection + source authority + access context`; selection may share internally, but wire/evidence shares only after compatibility proof. | **TARGET.** Current public query identity is effectively filter-only. |
| 19 | **Secret material made part of durable event storage** | Rust persistence stores obligations and expected pubkeys, never raw signing secrets. Standard platform providers hold secrets behind secure-storage capability boundaries. | **PARTIAL.** NIP-46 connection secrets stay in zeroizing session memory and durable reattachment persists only the expected pubkey/opaque identity reference; standard Keychain/Keystore providers and local-key zeroization hardening remain. |
| 20 | **Destructive replaceable edit composed or accepted from a stale base** | A protocol operation first establishes a source-scoped canonical base, preserves fields it does not own, and carries that exact event id into the store's atomic acceptance transaction. A changed winner produces typed `ReplaceableConflict` with no receipt, journal, or pending-row residue. Raw native writes cannot mint the guard. | **BUILT for the generic acceptance guard and NIP-02 follow/unfollow.** Both stores prove exact/none/conflict behavior; a real direct-Rust/iOS-FFI relay parity test proves acquisition, follow, no-op, preservation, and unfollow. Other replaceable protocols must define their own evidence and preservation policy. |
| 21 | **Live persistent store deleted by in-process reset** | `RedbStore` itself owns a canonical-target RAII registration from uninterrupted pre-open resolution/create/post-open resolution through final database drop. Reset takes the same store-layer registry mutex through target resolution, live check, and file removal; facade, `from_parts`, raw-engine, relative, existing-symlink, and dangling-final-symlink paths cannot bypass it. | **PARTIAL (#489 repaired source checkpoint).** Deterministic store tests drive both open/reset interleavings, exact two-owner refcount release, poison recovery, stable aliases, and resolved-target deletion. Rust/FFI plus Swift/Kotlin source falsifiers cover typed refusal with live bytes intact, Engine shutdown/drop, constructor/open/spawn failure release, raw `EngineThread`, `from_parts`, closed/idempotent reset, and native projection. Execution awaits the serialized build window. Cross-process locking and arbitrary external symlink retargeting are explicit known gaps, not claimed by this invariant. |
| 22 | **Route-override without ownership / ownership collision** | A `RoutePolicy` is representable only inside a `KindClaim` (no standalone registration API exists). Overlapping exclusive claims refuse to fold: `nmp_ownership::ClaimSet::build` returns a typed `ClaimOverlap` naming both owners and a witness kind, and the `nmp-audit` workspace test folds every enrolled module's `claims()` through it — enrollment is cargo-metadata-derived (a claim-bearing crate that forgets to enroll is a red build), so a collision is red workspace-wide even between modules no app links. `discovery_ack` consistency is enforced in both directions. | **PARTIAL (#521).** The fold, the workspace audit, route-authority ⊆ ownership, and discovery-ack checks are BUILT with fixture falsifiers (`crates/nmp-audit`, `nmp-ownership::claim_set`). The engine-construction wiring and the runtime publish gate ("a *policy-bearing* owned kind cannot be routed by the default policy" — an exclusive claim with `route_policy: None` routes default by the owner's own declaration, per the §4.2 layer-3 clarification) remain TARGET until Unit E claim-table routing — today's real claims (nip02 kind:3, nip51 kind:10009) declare `route_policy: None`, so no policy exists for the default path to violate; the `Automatic`-refusal falsifier lands with the gate. |

Every status must remain honest. A design document, issue, or passing adjacent
test is not proof. Promoting `TARGET`/`PARTIAL` to `BUILT` requires a falsifier
against the supported public facade and, where applicable, restart and platform
projection tests.

PR review applies this same rule prospectively, before a new bug class ever
needs a row here: see `docs/design/architecture-review-gates.md` for the six
review gates (Noun, Reachability, Bool-Lifecycle, Destructive-API, Cross-SDK
Parity, Falsifier-honesty) this ledger's rule expands into at review time.
