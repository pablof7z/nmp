# Sketch Manifest

## Design Direction

The NMP UI Gallery should feel like a polished native iOS product first and a
protocol falsifier second. Real component previews lead; scoped routing and
semantic evidence stay one deliberate interaction away. The design language is
editorial, content-first, and recognizably Apple-platform without imitating a
web dashboard or hiding difficult states.

## Reference Points

- SwiftUI and iOS native navigation, context menus, sheets, Dynamic Type, and
  accessibility behavior
- shadcn's browsable, app-owned component catalogue concept
- the breadth of `/Users/pablofernandez/Work/NDK-nhlteu/svelte/registry`
- NMP issue #75 and `docs/design/ui-components-strategy.md`
- the existing NMP Falsifier's real two-indexer routing and diagnostics proof
