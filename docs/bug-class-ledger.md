# Bug-class acceptance ledger

This is the replacement for the old repo's principle corpus + doctrine-lint + audit treadmill. Each entry names a **Nostr bug** and the **structural mechanism** ‚Äî a property of types or API surface ‚Äî that excludes it.

**Falsification standard.** To claim an entry holds, an agent attempts to *write the bug* against the public surface and records why the attempt cannot compile, cannot reach the wire, or cannot corrupt state. **Lints are not admissible mechanisms.** These falsification tests live in CI; an entry whose mechanism erodes (e.g. a new API adds a `Vec<RelayUrl>` route-override anywhere) is a red build.

The ledger is append-only in spirit: a newly discovered bug class demands a *surface change* plus a new entry ‚Äî never a lint.

| 5 | **Dedup or provenance loss** | Insert merges provenance on duplicate id before any other processing; provenance is a field of the stored row. Ids/signatures never re-derived post-verification. | 9 | **Enqueue treated as converged** | Write acceptance returns a receipt whose status is a stream with per-relay terminal acks; no *durable* publish API returns `void`/`bool`. | 11 | **App owning interest expansion** | `Derived` bindings resolve inside the engine; the query API returns final rows, never expanded intermediate sets; `Selector` is closed. | 12 | **Presentation in core** | Engine emits raw tokens only ‚Äî hex pubkeys, Unix timestamps, verbatim kind:0. No display helper in the engine crate; no formatted-string field on any FFI type. Formatting is unreachable because the vocabulary to express it is absent, not because a lint forbids it. **M0 amendment ‚Äî scope:** applies to **unencrypted** content. Encrypted payloads (NIP-17 gift-wrap, private NIP-51 items) are decrypted by an engine-internal encrypt/decrypt capability co-located with the signer (the key lives in the engine, not the app ‚Äî else identity-as-input breaks), and the engine emits the decrypted *raw tokens*, still doing zero presentation. Decryption is a capability, not a third noun. | not yet |

| 13 *(candidate)* | **Late-arriving old event skipped by pagination** | Two distinct cursors, kept apart by type: a **source cursor** over the durable ingest sequence (so an old-timestamped event that arrives late is still ingested and fanned out), and a **presentation cursor** over the ordered row collection (sort-key + id). Paging the view never advances acquisition, and acquisition never assumes the view's ordering. | candidate ‚Äî pending Tier-A at M4 |

Every entry is `not yet` until its milestone builds the mechanism and its falsification test is green in CI. This table is a second truth anchor: it never claims a mechanism holds before the test proves it. Entries marked *candidate* are provisional design outcomes not yet ratified by their gate.
