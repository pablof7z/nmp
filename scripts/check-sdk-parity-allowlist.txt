# Cross-SDK parity allowlist for scripts/check-sdk-parity.sh (gate 5, #496).
#
# Each entry suppresses EXACTLY ONE concept word on EXACTLY ONE side. This is
# the narrow escape hatch for a documented, intentional per-platform modeling
# difference -- it is not a mute button. An entry needs a justification that a
# reviewer can verify against the named source locations; if you cannot write
# one, the report line is a real parity gap and the fix belongs in the SDK,
# not here. The checker validates this file's format and fails on a malformed
# line, and reports (without failing) any entry that no longer suppresses
# anything, so stale entries surface for deletion.
#
# Format: <concept-word> <swift|kotlin> <justification, one line>
# - <concept-word>: lowercase token exactly as the checker reports it.
# - <side>: the single SDK side the word may be absent from.
#
# No current exceptions.
